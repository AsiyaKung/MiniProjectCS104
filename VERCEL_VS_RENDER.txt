======================================
⚠️ VERCEL DEPLOYMENT - ข้อควรรู้
======================================

Vercel ทำงานอย่างไรกับ Flask:

Vercel เป็น platform serverless ที่ดีที่สุด
สำหรับ Node.js/Next.js/Static sites

ทำงานได้บ้างกับ Python Flask แต่มีข้อจำกัด:
❌ Database persistence: ยาก (ทุกครั้งที่ deploy เริ่มใหม่)
❌ Session management: ไม่มี persistent storage
❌ Cold start: ช้ากว่า traditional hosting
✅ Free tier ที่ดี
✅ Auto deploy จาก GitHub

======================================
💡 แนะนำ: ใช้ RENDER แทน
======================================

Render.com ดีกว่า Vercel สำหรับ Flask:
✅ Support Python Flask โดยตรง
✅ Persistent database
✅ Free tier ที่ดี
✅ Auto deploy จาก GitHub
✅ Environment variables
✅ ไม่มี cold start issues
✅ Simple deployment

======================================
🚀 OPTION 1: Deploy ใน RENDER (แนะนำ)
======================================

STEP 1: สร้าง render.yaml

ในโปรเจค create ไฟล์: render.yaml

---

STEP 2: สร้าง Render Account

ไปที่: https://render.com
Login ด้วย GitHub

---

STEP 3: Create New Web Service

Dashboard → New → Web Service
→ เลือก Repository: MiniProjectCS104
→ เลือก Branch: main

Build Command:
pip install -r requirements.txt

Start Command:
gunicorn app:app

Environment: Python 3.11

---

STEP 4: Deploy

Click "Deploy"
รอ 2-3 นาที

ได้ URL: https://[project-name].onrender.com

---

STEP 5: ตั้งค่า Database

Database จะใช้ SQLite บนระบบ
ข้อมูลจะหายหลังจาก redeploy

ถ้าต้องเก็บข้อมูล:
→ ใช้ PostgreSQL (ฟรีได้ Render)

======================================
🔧 OPTION 2: Deploy ใน VERCEL
======================================

ถ้าต้องใช้ Vercel:

STEP 1: Update files

ก็เพิ่ม:
- vercel.json (ทำเสร็จแล้ว)
- api/index.py (ทำเสร็จแล้ว)

STEP 2: Push ขึ้น GitHub

git add .
git commit -m "Add Vercel deployment config"
git push origin main

STEP 3: Connect to Vercel

ไปที่: https://vercel.com
Login ด้วย GitHub
Import MiniProjectCS104 project

STEP 4: Deploy

Vercel จะ auto deploy

ได้ URL: https://[project-name].vercel.app

---

⚠️ ข้อจำกัด Vercel:
- Database: หายหลังจาก 60 นาที (cold storage)
- ต้องใช้ AWS S3 หรือ MongoDB Atlas
- Session management: ต้อง external service

======================================

======================================
